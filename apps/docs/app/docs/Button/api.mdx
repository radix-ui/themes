# API

Use the Button component to trigger actions throughout your application. The component provides six visual variants to match your interface context, four sizes for different interface densities, and built-in tooltips for enhanced user guidance. The component automatically handles icon sizing, supports responsive layouts, and provides accessibility compliance out of the box.

## Import

<CodeBlock buttonsPosition="center">

```tsx
import { Button } from '@kushagradhawan/kookie-ui';
```

</CodeBlock>

## Variants

Use the `variant` prop to select the visual style that best matches your interface context. Use `solid` for primary actions that users should notice first, `outline` for secondary actions that support the primary action, and `ghost` for utility functions that don't compete for attention. Reserve `classic` for premium experiences where you want to convey sophistication. For content-heavy interfaces, prefer `soft` and `surface` variants as they integrate more naturally with surrounding content.

## Sizes

Use the `size` prop to select the button size that matches your interface density. Use size `1` (24px) for toolbars and dense interfaces where space is limited, size `2` (32px) for standard interface contexts, size `3` (40px) for important actions and calls-to-action, and size `4` (48px) for hero sections where you want maximum visual impact. For mobile interfaces, prefer size `3` or `4` to ensure adequate touch targets.

The `size` prop supports responsive objects to adapt button sizing across different breakpoints. Use the format `{ initial: '1', sm: '2', md: '3', lg: '4' }` for responsive sizing.

<CodeBlock
  preview={
    <Flex gap="2" align="center" wrap="wrap">
      <Button variant="solid" size="1">Button</Button>
      <Button variant="solid" size="2">Button</Button>
      <Button variant="solid" size="3">Button</Button>
      <Button variant="solid" size="4">Button</Button>
    </Flex>
  }
>

```tsx
{/* Use size 1 for toolbars and dense interfaces */}
<Button size="1">Button</Button>

{/* Use size 2 for standard interface contexts */}
<Button size="2">Button</Button>

{/* Use size 3 for important actions and mobile touch targets */}
<Button size="3">Button</Button>

{/* Use size 4 for hero sections and maximum impact */}
<Button size="4">Button</Button>
```

</CodeBlock>

## Colors

Use the `color` prop with semantic colors to communicate the action's intent to users. Choose `green` for success actions like confirmations and approvals, `red` for destructive actions like deletions and cancellations, `amber` for warnings that require user attention, `blue` for informational actions like "Learn more" or "View details", and `gray` for neutral actions like "Cancel" or "Back".

### High Contrast

Set the `highContrast` prop to `true` when you need to ensure maximum visibility, particularly for users with visual impairments or when displaying buttons over complex backgrounds. Use this mode sparingly, as it can make interfaces feel heavy if overused.

For elevated interfaces with translucent materials, always use `highContrast` to maintain readability and ensure proper visual separation from the background.

<CodeBlock
  preview={
    <Flex gap="2" align="center">
      <Button color="gray" size="3">Button</Button>
      <Button color="gray" size="3" highContrast>Button</Button>
    </Flex>
  }
>

```tsx
<Button color="gray" size="3">Button</Button>
<Button color="gray" size="3" highContrast>Button</Button>
```

</CodeBlock>

## Material

Use the `material` prop to control how buttons render their visual elements. Choose `solid` for opaque backgrounds and borders, or `translucent` for elevated effects that create depth and visual separation from the background.

For translucent effects to be visible, buttons must appear over images or dynamic backgrounds. Avoid using `outline` and `ghost` variants in translucent contexts, as they may not provide sufficient contrast and visibility.

## States

### Loading

Set the `loading` prop to `true` to show a spinner and disable interaction during asynchronous operations. The component automatically sets `disabled` to `true` when `loading` is `true`, preventing multiple submissions and providing clear visual feedback that the action is in progress.

You don't need to manually set `disabled` when using `loading` - the component handles this automatically.

<CodeBlock
  preview={
    <Flex gap="2" align="center">
      <Button size="3" color="gray" highContrast>Submit</Button>
      <Button size="3" color="gray" highContrast loading>Submitting...</Button>
    </Flex>
  }
>

```tsx
<Button size="3" color="gray" highContrast>Submit</Button>
<Button size="3" color="gray" highContrast loading>Submitting...</Button>
```

</CodeBlock>

### Disabled

Set the `disabled` prop to `true` to prevent interaction when an action is currently unavailable. Common use cases include form validation states, when required fields are empty, or when an action requires user authentication.

Always provide clear visual feedback about why a button is disabled, either through tooltips or surrounding text.

<CodeBlock
  preview={
    <Flex gap="2" align="center">
      <Button size="3">Save</Button>
      <Button size="3" disabled tooltip="Please fill all required fields">Save</Button>
    </Flex>
  }
>

```tsx
<Button size="3">Save</Button>
<Button size="3" disabled tooltip="Please fill all required fields">Save</Button>
```

</CodeBlock>

## Layout

### Full Width

Set the `fullWidth` prop to `true` when you want the button to expand and fill its container. This is particularly useful for mobile layouts and form submissions where you want to maximize the touch target area and create a clear visual hierarchy.

<CodeBlock
  preview={
    <Flex direction="column" gap="2" width="300px">
      <Button size="3" color="blue" fullWidth>Complete</Button>

</CodeBlock>

### Flush

Set the `flush` prop to `true` with the `ghost` variant to remove visual padding for seamless text integration. This is useful when you want the button to appear as inline text while maintaining its interactive behavior.

</CodeBlock>

## Margin

Use margin props to control spacing around buttons. All margin props are responsive and support both design system tokens and CSS string values.

## Tooltips

Use the `tooltip` prop to provide additional context for button actions, especially for icon buttons or actions that might not be immediately clear to users. Tooltips appear on hover and focus, making them accessible to both mouse and keyboard users. For icon-only buttons, always provide a tooltip to explain the action.

## Polymorphism

### As

Use the `as` prop to render buttons as different HTML elements while maintaining button styling. This is useful for navigation links, routing integration, or custom interactive elements that need button appearance.

<CodeBlock
  preview={
    <Flex gap="2" align="center" wrap="wrap">
      <Button size="3" as="a" href="/home">Go Home</Button>
      <Button size="3">Button</Button>
      <Button size="3" as="a" href="/profile">View Profile</Button>
    </Flex>
  }
>

```tsx
<Button size="3" as="a" href="/home">Go Home</Button>
<Button size="3">Button</Button>
<Button size="3" as="a" href="/profile">View Profile</Button>
```

</CodeBlock>

### AsChild

Use the `asChild` prop to merge button props with a child element using the Radix Slot pattern. This is ideal for composition with routing libraries like Next.js Link or React Router, as it applies button styling without creating an additional wrapper element.

<CodeBlock
  preview={
    <Flex gap="4" align="center">
      <Button size="3" asChild><a href="/dashboard">Go to Dashboard</a></Button>
      <Button size="3" variant="outline" asChild><a href="/profile">View Profile</a></Button>
    </Flex>
  }
>

```tsx
<Button size="3" asChild><a href="/dashboard">Go to Dashboard</a></Button>
<Button size="3" variant="outline" asChild><a href="/profile">View Profile</a></Button>
```

</CodeBlock>

## Responsive

Use responsive objects with the `size` prop to adapt button sizing across different breakpoints. The component uses a mobile-first approach where each breakpoint applies styles at its minimum width and above.

<CodeBlock
  preview={
      <Button size={{ initial: '1', sm: '2', md: '3', lg: '4' }}>Button</Button>
  }
>

```tsx
<Button size={{ initial: '1', sm: '2', md: '3', lg: '4' }}>Button</Button>
```

</CodeBlock>

**Breakpoints:**
- `initial` - Base styles (default)
- `xs` - 520px and above
- `sm` - 768px and above
- `md` - 1024px and above
- `lg` - 1280px and above
- `xl` - 1640px and above
