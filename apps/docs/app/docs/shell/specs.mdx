### Animation Principles

<Callout.Root variant="soft" size="2" color="blue" my="7">
  <Callout.Text>The Shell uses a two-phase animation system to prevent content from appearing compressed during transitions.</Callout.Text>
</Callout.Root>

**Collapse Sequence:**
1. Content fades out immediately (150ms)
2. Container animates to collapsed state (250ms, delayed 150ms)
3. Total duration: 400ms

**Expand Sequence:**
1. Container animates to expanded state immediately (250ms)
2. Content fades in after container settles (150ms, delayed 250ms)
3. Total duration: 400ms

### Memory Usage

- **Base overhead**: ~2KB CSS, ~5KB JavaScript
- **Runtime state**: ~1KB per shell instance
- **Event listeners**: Minimal (resize, media queries only)
- **DOM nodes**: 8-15 nodes per shell (depending on configuration)

### Optimization Recommendations

<Callout.Root variant="soft" size="2" color="yellow" my="7">
  <Callout.Text>For optimal performance, avoid frequent programmatic size changes and minimize content updates during animations.</Callout.Text>
</Callout.Root>

**Best practices:**
- Use CSS transforms for non-layout animations
- Debounce resize event handlers
- Implement virtual scrolling for large content lists
- Avoid complex calculations during resize operations
- Use `will-change: transform` for animated elements
